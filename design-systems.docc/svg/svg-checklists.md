Checklists for creating and reviewing SVGs.

## Creation Checklist
- [ ] Does the SVG have a descriptive title?
- [ ] Is all CSS contained within a `<style>` block?
- [ ] Are CSS variables used for theming?
- [ ] Are related elements grouped with `<g>` tags?
- [ ] Do all targeted elements have descriptive IDs?
- [ ] Does the SVG use the design system's color palette?
- [ ] Is the stroke width consistent?
- [ ] Is the corner radius consistent?
- [ ] Does the SVG use the design system's font?
- [ ] Is the animation short and meaningful?
- [ ] Does the animation use easing functions?
- [ ] Does the `@Image` tag have descriptive `alt` text?
- [ ] Are ARIA attributes used for accessibility?
- [ ] Is the color contrast sufficient?
- [ ] Is the SVG optimized for file size?
- [ ] Is the filename meaningful and in kebab-case?
- [ ] If generated, does the filename have the correct suffix?
- [ ] Is the source file (if any) version-controlled?

## Review Checklist
- [ ] Does the SVG adhere to all the principles and patterns of the SVG design system?
- [ ] Is the SVG visually appealing and easy to understand?
- [ ] Is the SVG accessible to people with disabilities?
- [ ] Is the SVG performant?
- [ ] Is the SVG code well-structured and easy to maintain?
